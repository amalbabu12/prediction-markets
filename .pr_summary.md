# PR Summary

## What Changed

### `backtest_arbitrage.py`
- Added a real HuggingFace NLI backend (`HuggingFaceNLIBackend`) using `typeform/distilbert-base-uncased-mnli` for pair classification. The model uses entailment scoring (ENTAILMENT/CONTRADICTION probabilities) to determine whether two market questions resolve the same way.
- The script now accepts an optional `--mock` flag: without it, the real HF model is used; with it, the fast deterministic mock runs instead.
- Output file is now mode-dependent: `backtest_results.json` (mock) vs `backtest_results_hf.json` (real HF).

### New Files Added
| File | Description |
|------|-------------|
| `BACKTEST_REPORT.md` | Full report: strategy overview, script summary, complete results tables for both mock and HF runs, and a written comparative analysis |
| `backtest_results_hf.json` | Raw JSON results from the real HuggingFace NLI run |
| `backtest_output_hf.txt` | Console output from the real HuggingFace NLI run |

### Unchanged Files
- `backtest_results.json` — original mock LLM results (kept for comparison)
- `backtest_output.txt` — original mock LLM console output (kept for comparison)

## Real-LLM Run Results

**Model:** `typeform/distilbert-base-uncased-mnli` (DistilBERT fine-tuned on Multi-Genre NLI, 66 M params, CPU inference)

| Metric | Mock LLM | HuggingFace NLI |
|--------|----------|-----------------|
| Same-outcome pairs classified | 32 / 32 | 24 / 32 |
| Overall accuracy | 46.9% | **62.5%** |
| False positives | 17 | **10** |
| True negatives | 0 | **6** |
| Trades executed | 54 | **38** |
| Pair match rate | 48.1% | **60.5%** |
| Total net P&L | −2.6536 | **−2.3717** |

The real NLI model improves pair classification accuracy by 15.6 percentage points and reduces false-positive trades by 7, directly improving total P&L by +$0.28. Both runs remain unprofitable because the payoff structure requires ~91%+ pair match rate to break even (one wrong trade costs ~20× a correct trade's gain).

## How It Was Tested

1. **Mock run (baseline):** `python3.11 backtest_arbitrage.py --mock` — deterministic, fast, produces `backtest_results.json`.
2. **Real HF run:** `python3.11 backtest_arbitrage.py` — downloads and runs `typeform/distilbert-base-uncased-mnli` on CPU, produces `backtest_results_hf.json`.
3. Results were verified by inspecting the console output (saved to `backtest_output_hf.txt`) and comparing confusion matrices, trade counts, and P&L figures between runs.
4. The NLI model correctly produces non-trivial TN=6 (pairs correctly rejected), confirming it is doing real semantic reasoning rather than always predicting same-outcome.
